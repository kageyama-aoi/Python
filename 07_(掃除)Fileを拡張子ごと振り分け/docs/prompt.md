あなたは Git / Python / pytest に詳しいアシスタントです。

以下の条件を満たす形で、ローカル Git フック（pre-push）を体験できる最小構成を作りたいです。

# 前提
- すでに Python ファイルがいくつか存在する Git リポジトリがある
- pytest は使用可能（インストール済み or pip install 可能）
- GitHub Actions などの CI はまだ考えない
- ローカルで「push できない体験」をしたい

# やりたいこと
- git push 時に pytest を実行する
- テストが失敗したら push を中断する
- 成功したら push を続行する

# やってほしいこと
1. 既存の Python ファイルを壊さずに、
   「必ず最初は失敗する」仮のテストコードを1つ作成してください
2. `.git/hooks/pre-push` に置くシェルスクリプトの内容を提示してください
3. chmod を含めた実行手順をステップ形式で説明してください
4. push したときに「どんな表示が出るか」を例として書いてください
5. 最後に、この仕組みが Git 的にどう動いているかを簡潔に説明してください

出力は以下の構成でお願いします。

- 仮のテストコード例（pytest）
- pre-push スクリプト全文
- 実行手順（コマンド付き）
- push 失敗時の出力例
- 仕組みの解説（短く）
