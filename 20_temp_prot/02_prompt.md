あなたは業務向けWebポータルのUI/UXデザイナーです。
以下の「データフローポータル（DB更新フロー可視化）」の画面について、
情報の視認性と理解速度を上げるための見た目改善を提案してください。

【前提】
- 利用者はエンジニア・業務設計者
- 正確さと「一瞬で把握できること」を最重視
- 色数は増やしすぎない（既存の added / removed / changed / same は維持）

【改善したいポイント】
1. 横軸のテーブル名（products / shipments / subscriptions）を
   「この画面の主役」として明確に強調したい
2. case_id / trigger / operation / table は補助情報として一段弱めたい
3. 更新内容（値の変化）が自然に目に入る視線誘導を作りたい
4. 長時間見ても疲れない業務画面にしたい

【具体的に検討してほしいこと】
- フォントサイズ・太さ・大文字/小文字の使い分け
- テーブル見出し行とデータ行のコントラスト設計
- 横軸（テーブル単位）をブロックとして認識させる工夫
- 行・列の余白（padding / line-height）の最適化
- 「変化があるセル」と「説明用セル」の視覚的分離

【制約】
- 情報量は減らさない
- 新しいUIコンポーネントは追加しない
- CSSで実現できる範囲の改善に限定

改善案は、
- 何をどう変えるか
- なぜそれが有効か（認知・視線の観点）
をセットで説明してください。
