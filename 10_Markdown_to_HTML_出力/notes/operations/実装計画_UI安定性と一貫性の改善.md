# 実装計画: UI安定性と一貫性の改善

## 目的

- 画面遷移時の「横幅ズレ」「ちらつき感」を減らし、利用時ストレスを下げる。
- `/`（メタ編集）・`/import`（取込）・`/kb`（一覧）の見た目と操作感を統一する。

## 参照ドキュメント

- `docs/UI安定性と一貫性デザイン改善ガイド.md`

## 対象範囲

- `app.py` のHTML/CSS（`/` と `/import`）
- `build.py` の `index.html` 生成テンプレート（`/kb` 相当）
- 必要に応じて `html/style.css` または新規 `html/shared_ui.css`
- `README.md`（運用変更がある場合のみ）

## スコープ外

- 情報設計そのものの大幅変更（項目追加・機能追加）
- バックエンドAPI仕様の変更

## 成果物

1. 共通トークン（幅・余白・タイポ）の定義
2. 3画面共通のヘッダー/ナビスタイル
3. フォーム部品の寸法統一
4. メッセージ表示領域の高さ固定（レイアウトジャンプ抑制）
5. 確認結果メモ（本ファイル末尾に追記）

## 実装ステップ

## Step 1: 現状差分の固定

- 3画面の現状値を洗い出し
  - コンテナ幅
  - 余白
  - フォントサイズ
  - ブレークポイント
- 基準値を決定
  - `--container-max`（候補: 1120px）
  - `--space-*`
  - `--radius-*`

## Step 2: 共通スタイル導入

- 共通CSSを定義（既存 `html/style.css` 拡張 or `html/shared_ui.css` 新設）
- 以下を共通クラス化
  - ページコンテナ
  - ヘッダー/ナビ
  - ツールバー
  - フォームコントロール
  - ステータスメッセージ領域

## Step 3: メタ編集画面（`/`）適用

- `app.py` 内のCSSを共通化方針に沿って更新
- 既存UI機能（検索・一括編集・保存）を壊さないことを確認

## Step 4: 取込画面（`/import`）適用

- `/` と同じ骨格にそろえる
- カテゴリ/タグ候補UIの見た目密度を統一
- 保存結果表示で上下ジャンプが発生しないよう固定領域化

## Step 5: 一覧画面（`/kb` / `build.py`生成）適用

- `build.py` テンプレートの寸法・余白を共通化
- 生成後の `html/index.html` で見た目が一致することを確認

## Step 6: 検証

- 画面幅: `360 / 768 / 1280`
- ズーム: `200% / 400%`
- 操作: 保存前後・検索時・タグ操作時のレイアウト安定
- CLI確認:
  - `python -m py_compile app.py`
  - `python build.py`

## 受け入れ条件

- 3画面でコンテナ幅・余白・部品寸法が実質統一されている
- 保存/更新操作で主要UIの位置が大きく跳ねない
- 既存機能（メタ編集・取込保存・ビルド生成）が回帰していない
- モバイル幅でも崩れず操作可能

## リスクと対策

- リスク: `app.py` と `build.py` が別経路でHTMLを出しているため再分岐しやすい  
  対策: トークン名と値を同一化し、差分レビュー項目に追加

- リスク: インラインCSSの残存で部分的に古い見た目が残る  
  対策: 実装完了時に「共通化漏れチェック」を実施
