## 詳細（上位）

### 1. 配布マニフェスト自己同梱
- 現状:
  - `scripts/deploy_toolkit.ps1` は `scripts/toolkit_manifest.json` 前提。
  - ただし配布対象 `files` にマニフェスト自身が入っていないため、コピー先で再配布が壊れる。
- 対応:
  - `scripts/toolkit_manifest.json` の `files` に `scripts/toolkit_manifest.json` を追加。
- 期待効果:
  - ツールキットの再利用性が上がり、コピー先でも同じ運用が可能。

### 2. `style.css` 同梱
- 現状:
  - 各ページHTMLは `style.css` を参照。
  - 配布対象に `html/style.css` が無い。
- 対応:
  - `scripts/toolkit_manifest.json` に `html/style.css` を追加。
  - もしくは `build.py` で `style.css` 未存在時に最小デフォルトCSSを自動生成。
- 期待効果:
  - 「ページは開くが見た目が崩れる」問い合わせを減らせる。

### 3. `start_here.ps1` のパス解決統一
- 現状:
  - `start_here.ps1` は `$MyInvocation.MyCommand.Path` 依存。
  - 同様の実装で以前 `run_build.ps1` が null エラーになった履歴あり。
- 対応:
  - `run_build.ps1` と同じ `Resolve-ScriptPath` 関数へ統一。
- 期待効果:
  - 起動入口の安定化。再発防止。

### 4. デプロイ時のエラーメッセージ改善
- 現状:
  - ドライブ権限エラー時にPowerShell例外がそのまま出る。
- 対応:
  - `Ensure-Dir` / `Test-Path` / `Copy-Item` を `try/catch` で包み、  
    `対象パス` `失敗理由` `次アクション` を短く表示。
- 期待効果:
  - 運用者が自力で復旧しやすい。

### 5. Git漏洩リスク低減（MD運用）
- 現状:
  - 本命MDをこの基盤に置かない運用だが、誤配置時の保護が薄い。
- 対応案（どちらか）:
  - A: `.gitignore` で `md/*.md` を除外し、サンプルだけ `!md/sample_*.md` を許可
  - B: pre-commit相当チェック（`md/` に本命命名規則があれば警告）
- 期待効果:
  - 人為ミス時の漏洩確率を下げる。
