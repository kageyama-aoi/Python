# Issue下書き: AI生成Markdownナレッジの閲覧性改善

## タイトル案   
`AI生成Markdownナレッジの分散を抑え、閲覧導線を改善する`

## 本文（そのまま貼り付け可）
```md
## 背景
生成AIで作成したMarkdownと図版（画像）が増えるにつれ、情報が分散し、必要な情報に到達しにくくなっている。

## 現在の課題
- ツールは「変換機能」はあるが、閲覧体験の設計意図が伝わりにくい
- ドキュメントの粒度・配置ルールが未整理で、内容が散らばる
- 図版の管理ルール（命名/配置/参照）が弱く、追跡しにくい
- インデックスはあるが、運用方針（何をどこへ集約するか）が不足

## 目的
- 分散したMarkdown/図版を、`html/index.html` 起点で探索しやすい知識ベースにする
- 新規作成時に迷わない運用ルールを定義する
- 継続的に増えても閲覧性が落ちない状態を維持する

## 受け入れ条件
- README冒頭で「何のためのツールか」が1読で伝わる
- ドキュメント作成テンプレートを使った作成手順が定義されている
- 画像/リンク記法の作成手間が補助ツールで削減されている
- ドキュメントと図版の命名・配置ルールが定義されている

## タスク案
- [ ] READMEに「用途」「想定運用フロー」「対象ユーザー」を明記
- [ ] `docs/template.md` 運用ルールをREADMEに追記
- [ ] 図版の配置ルール（例: `picture/YYYYMMDD_topic_xxx.png`）を決定
- [ ] Front Matter運用ルール（category/tags）を定義
- [ ] 定期メンテ項目（未分類、リンク切れ、孤立画像）をチェックリスト化

## 関連ファイル
- `README.md`
- `docs/template.md`
- `md_helper.ps1`
- `build.py`
```

## Issueコメント案（ブラッシュアップ用）

### コメント1: 何が弱いか（厳しめ）
```md
現状は「便利な変換ツール」にはなっているが、「知識を読むためのプロダクト」にはまだ到達していない。
特に不足しているのは、機能追加よりも情報設計（粒度・導線・命名規約）。
```

### コメント2: 改善の優先順位
```md
優先順位:
1. 入口（READMEの用途定義）
2. 作成時（テンプレート + md_helper運用）
3. 保守時（未分類/孤立画像/リンク切れ検知）
```

### コメント3: Doneの定義
```md
「Done」は機能実装ではなく、利用者が目的情報に短時間で到達できること。
目安として、初見ユーザーが3クリック以内で関連ページに到達できる状態を目標にする。
```
