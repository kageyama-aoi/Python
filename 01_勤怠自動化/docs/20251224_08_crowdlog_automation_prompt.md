# CrowdLog 自動化開発用プロンプト

このドキュメントは、CrowdLogの工数実績ダウンロードを自動化するために、AI（私）に渡す指示プロンプトです。
`docs/` フォルダに対象画面のHTMLソースファイル（例: `crowdlog_source.html`）を配置した状態で使用してください。

---

## AIへの指示プロンプト

以下のテキストをコピーして、AIに送信してください。
※ `[配置したHTMLファイル名]` の部分は、実際に置いたファイル名（例: `target_page.html`）に書き換えてください。

```markdown
現在、新しい自動化プロジェクトディレクトリにいます。
このプロジェクトは、汎用的なWeb自動化ツールのひな形（Python + Selenium）をベースにしています。

今回自動化したいのは **CrowdLogの工数実績ダウンロード** です。
ターゲットURL: `https://app.crowdlog.jp/download/?rm=timesheet`

### 前提情報
`docs/` ディレクトリ配下に、操作したい画面のソースコードファイル HTML を配置しました。

### 実現したい機能
1.  **期間の入力**:
    *   「期間開始日」フィールドに、**実行時の「今月の1日」** (例: 2025-12-01) を自動計算して入力する。
    *   「期間終了日」フィールドに、**実行時の「今月の末日」** (例: 2025-12-31) を自動計算して入力する。
2.  **ダウンロード実行**:
    *   画面上の「ダウンロード」ボタンをクリックして、CSVファイルを出力する。
3.  **ファイル操作**:
    *   ダウンロードされたCSVファイルを、デフォルトのダウンロードフォルダから、任意の保存先（例: `data/downloads/`）に移動する。

### 依頼事項

**Step 1: 画面解析と設定ファイルの更新**
1.  `docs/[配置したHTMLファイル名]` を読み込んで解析し、以下の要素のセレクタ（`name`, `id`, `xpath` 等）を特定してください。
    *   期間開始日の入力フィールド
    *   期間終了日の入力フィールド
    *   ダウンロードボタン
2.  特定した情報をもとに、`config/config.yaml` を書き換えてください。
    *   `app.url` の更新。
    *   `fields` セクションに日付フィールドを追加。
    *   `selectors` セクションにボタンを追加。

**Step 2: ロジックの実装**
1.  **日付計算**: `src/form_handler.py` (または適切な場所) を修正し、Pythonの `datetime` モジュールを使って「今月の初日」と「末日」を動的に計算するロジックを追加してください。計算した値をフォーム入力値として使用するようにしてください。
2.  **ファイル移動**: ダウンロード完了を検知し、ファイルを移動する処理を追加してください（`src/file_utils.py` などを新規作成するか、`main.py` に記述するかはお任せします）。

まずは **解析結果** と **修正方針（プラン）** を提示してください。
```
