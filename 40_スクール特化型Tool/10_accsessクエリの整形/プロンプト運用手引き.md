# ER解析プロンプト集（v2）

このファイルは、同じ作業を高再現で依頼するためのプロンプト集です。  
用途に応じて、そのままコピーして使えます。

## 0. 共通前提（先頭に付ける）

```md
あなたはデータベースリバースエンジニアリング専門エンジニアです。
リポジトリ内の既存成果物を再利用し、推測は根拠付きで行ってください。

必須:
- 最新ディレクトリ（YYYYMMDD_HHMMSS の最大値）だけを解析対象にする
- JOIN根拠を最優先する
- 不確実な推定はコメントで残す
- 出力ファイルを明示する
```

## 1. 全体ER（full）を作る

```md
目的:
output/split_sql 配下の最新SQL群から、全体ERを Mermaid で生成してください。

入力:
- output/split_sql/<latest>/*.sql

出力:
- output/er_diagram_latest.mmd

要件:
- ```mermaid / erDiagram``` 形式
- テーブル名、主要カラム、JOIN由来の関係線
- PK/FKは候補として明示
- 不確実な関係は `%% uncertain:` で残す
```

## 2. 主要テーブルER（core）を作る

```md
目的:
全体ERから一時・作業・中間ノイズを除き、主要テーブル中心のERを作成してください。

出力:
- output/er_diagram_core.mmd

除外ヒント:
- prefix: Q, FIX_, TEMP_
- contains: 一時, 作業, 変更用, 削除, 復元

要件:
- 可読性優先（テーブル数を抑える）
- 関係線は業務的に意味のあるものを優先
```

## 3. 主要ERに代表列を追加する

```md
目的:
output/er_diagram_core.mmd を元に、各テーブルに意味が分かる代表列を2〜3個追加した版を作成してください。

出力:
- output/er_diagram_core_with_sample_columns.mmd

要件:
- 既存の関係線は維持
- 代表列は業務的説明性を優先（コード・IDだけに寄せない）
```

## 4. フラグ別ERを3本作る（受講生/講師/一般研修）

```md
目的:
T利用者のフラグ（受講生/講師/一般研修）ごとに、見るべきテーブルを分けたERを作成してください。

入力:
- output/er_diagram_core_with_sample_columns.mmd（core版）
- 必要に応じて output/er_diagram_latest.mmd（full版）
- output/30_docs/コンテキスト定義.md

出力（core版）:
- output/er_diagram_core_student.mmd
- output/er_diagram_core_instructor.mmd
- output/er_diagram_core_general_training.mmd

出力（full版）:
- output/er_diagram_full_student.mmd
- output/er_diagram_full_instructor.mmd
- output/er_diagram_full_general_training.mmd
```

## 5. 差分DIFを作る（最重要）

```md
目的:
3パターンERの差分（共通/固有）を Markdown で整理してください。

出力:
- core差分: output/core_diagram_variant_diff.md
- full差分: output/full_diagram_variant_diff.md

要件:
- 件数サマリ（テーブル数/リレーション数）
- 3版共通テーブル
- 各版のみのテーブル
- 3版共通リレーション
- 各版のみのリレーション
```

## 6. コンテキスト更新（継続運用）

```md
目的:
分析結果に基づいて コンテキスト定義.md を更新してください。

出力:
- コンテキスト定義.md（または output/30_docs/コンテキスト定義.md）

要件:
- 各ルールに status: 確定 / 推定 / 未定 を付与
- 新規フラグが出たら Flag Semantics を最優先更新
```

## 7. 整理・納品

```md
目的:
成果物を output 配下で整理し、最新採用版を output/00_latest に集約してください。

要件:
- output/README_OUTPUT.md を更新
- 旧単発ファイルは output/archive/cleanup_<timestamp>/ に退避
- 最終的に「まず見るべき5ファイル」を提示
```
