# [refactor] ビューアー基盤改善（例外処理・安全描画・操作性）

## 背景
業務ロジック タイムラインビューアーは実用段階に入ったが、保守性と運用安全性の観点で先に整備したい課題がある。  
特に「不正JSON入力時の復帰性」と「HTML描画時の安全性」は、運用時のトラブル回避のため優先度が高い。

## 課題
1. `JSON.parse` の例外未処理で、読み込み失敗時に画面が壊れる
2. `innerHTML` への未エスケープ文字列埋め込み箇所がある
3. `courses` が空のケースでユーザー向けメッセージがない
4. タブが `div + onclick` 実装のためキーボード操作性が低い
5. `assets/js/app.js` に責務が集中しており拡張時に破壊リスクが高い

## 目的
- 運用時の入力揺れ（壊れたJSON、欠損データ）に強いUIにする
- 安全な描画方式へ寄せ、将来の仕様追加を安全にする
- 今後の機能追加に備えて、JSの構成を段階的に整理する

## 受け入れ条件
- 不正JSON読込時、画面がクラッシュせずエラーメッセージを表示する
- 動的描画でユーザー入力由来文字列を直接 `innerHTML` に流さない
- `courses` が0件の場合、空状態の案内UIを表示する
- タブ切替がキーボード（Tab/Enter）で操作できる
- `app.js` の責務を最低2レイヤー（データ整形 / 描画）に分割する方針が入る

## 実装タスク案
- [ ] JSON読込処理に `try/catch` を追加し、失敗時の通知UIを実装
- [ ] 文字列描画を `textContent` ベースへ移行（必要箇所のみテンプレート許可）
- [ ] 空データ表示コンポーネントを追加
- [ ] タブを `button` 要素へ変更し、ARIA属性を設定
- [ ] `app.js` を段階分割（例: `state`, `render`, `format`）

## 備考
- 本Issueは「機能追加」ではなく「基盤品質改善」を目的とする
- 見た目の大幅変更は対象外（既存UIトーンは維持）
