# Repository Guidelines

このリポジトリは Python プロジェクトの標準構成を定義するテンプレート兼参照資料です。構成ドキュメントと新規プロジェクト生成スクリプトを含みます。以後のやりとり・文書は日本語で行う前提です。

## プロジェクト構成とモジュール整理

- ルート文書: `01_prompt.md` がサンプル仕様です（標準構成の説明を更新する場合はここも参照）。
- 生成スクリプト: `scripts/create_project.py` が標準構成の新規プロジェクトを作成します。
- 生成後の代表構成: `src/`（実装）、`config/`（設定）、`docs/`（設計メモ）、`data/`（入出力）、`logs/`、`tests/`、`scripts/`。
- 例: `src/handlers/` は業務ロジック、`src/utils/` は共通ヘルパー用。`config/` はコードから参照される唯一の設定源です。

## ビルド・テスト・開発コマンド

- 新規プロジェクト作成: `python scripts/create_project.py <New_Project_Name>`  
  標準構成の雛形を生成します。出力先はカレントディレクトリ配下です。
- このリポジトリ自体にビルド/テストはありません。生成物の README に、実行やテストのコマンドを明記してください。

## コーディング規約・命名規則

- PEP 8 準拠（4スペース、snake_case、モジュール名は短く明確に）。
- 設定値は `config/` に集約し、コードへの直書きを避けます。
- モジュール命名例: `config_manager.py`, `handlers/csv_loader.py`, `utils/log.py`。

## テスト方針

- 生成物には `tests/` を作成しますがフレームワークは未設定です。
- テストを追加する場合は `test_*.py` 形式で統一し、利用するフレームワーク（例: `pytest`）と実行方法を README に記載してください。

## コミット・プルリクエストの指針

- コミットは短く具体的な日本語要約。必要ならコロンで補足します。  
  例: `設定UIの操作性改善: ページ順制御の調整`
- プルリクエストには変更概要、関連ドキュメント更新、再現手順（生成スクリプト実行など）を含めてください。

## セキュリティ・設定の注意

- 機密情報は `.env` に保存し、Git 管理しません。
- 設定は `config/` を唯一の正とし、仕様変更時は必ず文書も更新します。
